# How programming works (and Python in cybersecurity)

---

## How programming works

**Programming** is a process for creating a **specific set of instructions** that a computer executes to perform tasks.

**Computer programs** are everywhere: laptops, phones, and many other devices run software that tells hardware what to do.

Many **programming languages** exist to write those instructions. **Python** is one of them.

### From human-readable code to what the CPU runs

Programming languages are ultimately represented as **binary** (sequences of **0s and 1s**) that map to operations the **central processing unit (CPU)** can perform—such as adding numbers or loading a value from memory.

Writing raw binary by hand would be slow and error-prone. Higher-level languages like **Python** let you express complex logic with **less syntax**, so you can instruct computers more efficiently than at the machine-code level.

---

## Using Python to program

**Python** is a **general-purpose** language. Examples of what it can be used for include:

- building websites  
- performing **data analysis**  
- **automating** repetitive workflows  

### Interpreter

Python source code is executed with help from an **interpreter**: a program that **translates** Python into runnable instructions, typically **line by line**, so the machine can run your script.

---

## Python versions

There are **multiple versions** of Python. This certificate track uses **Python 3**.

Always know **which version** you are running: **syntax** (the rules that define valid structure in the language) can differ between major versions.

---

## Python in cybersecurity

In cybersecurity, Python is often used for **automation**: using technology to **reduce manual effort** on **common, repetitive** tasks.

Examples of areas where Python may automate security work:

- **Log analysis**  
- **Malware analysis**  
- **Access control list** management  
- **Intrusion detection** (supporting workflows, parsing, integrations)  
- **Compliance** checks  
- **Network scanning**  

---

## Key takeaways

- **Programming** means writing instructions computers follow; programs drive most modern devices.  
- Languages like **Python** are easier for humans than raw **binary**; the CPU still executes low-level operations underneath.  
- Python runs through an **interpreter** (commonly line-by-line translation to execution).  
- Track your **Python version** (this course uses **Python 3**); **syntax** can change between versions.  
- In security, Python shines for **automation** across logs, malware work, ACLs, detection support, compliance, and scanning.
