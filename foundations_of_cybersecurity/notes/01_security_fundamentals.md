# Security fundamentals and the analyst role

## What a security / cybersecurity analyst does

- **Monitors networks** for breaches and suspicious activity.  
- **Helps develop strategies** to secure the organization.  
- **Researches IT security trends** to stay ahead of potential threats.  
- **Works to prevent incidents** before they spread or cause harm.

---

## Core concepts (vocabulary you will reuse everywhere)

### Compliance

Process of adhering to **internal standards** and **external regulations**. Strong compliance helps organizations avoid fines and reduce the chance of security breaches.

### Security frameworks

**Guidelines** used to build plans that mitigate risks and threats to data and privacy.

### Security controls

**Safeguards** designed to reduce specific security risks. Controls are implemented **with** frameworks to achieve a strong security posture.

### Security posture

An organization’s **overall ability** to defend critical assets and data **and** adapt when threats or the environment change. Better posture generally means lower risk.

### Threat actor (malicious attacker)

Any **person or group** that presents a security risk to computers, applications, networks, or data.

### Internal threat

Can be a **current or former employee**, **vendor**, or **trusted partner**. May be:

- **Accidental** — e.g., an employee clicks a malicious link in email.  
- **Intentional** — e.g., unauthorized data access or other risky behavior.

### Network security

Keeping the organization’s **network infrastructure** safe from unauthorized access—including data, services, systems, and devices on that network.

### Cloud security

Ensuring cloud-hosted assets are **configured correctly** and **only authorized users** can access them.

**Cloud (short definition):** A network of servers/computers in **remote data centers**, reachable over the internet, storing resources and data. Cloud security is a growing specialty focused on protecting data, applications, and cloud infrastructure.

## Why programming matters in security

Programming lets you give the computer a **specific set of instructions**. Common security uses:

- **Automation** of repetitive work (e.g., checking lists of malicious domains).  
- **Reviewing** web traffic.  
- **Alerting** on suspicious activity.

## Key tools and practices (intro level)

### Security information and event management (SIEM)

- Collects and analyzes **log data** (records of events such as unusual logins).  
- Helps analysts **monitor** critical activity across the organization.  
- Speeds up identification and analysis of **threats, risks, and vulnerabilities**.

### Intrusion detection systems (IDSs)

- Monitor **system activity** and raise **alerts** for possible intrusions.  
- Widely used to protect assets and data.  
- Example use: watch the network for signs of malicious activity such as **unauthorized access**.

### Threat landscape knowledge

Staying current on **threat actors**, **malware**, and **attack methods** so defenses match real-world tactics—e.g., recognizing a **new ransomware variant** early.

### Incident response

Following **documented policies and procedures** when something goes wrong.

**Example flow:** Alert suggests malware → analyst starts the organization’s **incident response process** → **investigate** root cause → **remediate** (fix and recover) per policy.
