# Programming, operating systems, and common security tools

## Programming

**Definition:** Creating instructions for a computer to perform tasks.

### Security analyst uses

- **Automation** — technology that reduces manual, repetitive work and **human error**.  
- Languages called out in course: **Python** (scripting/automation), **SQL** (databases).

### SQL and databases

- **Database** — organized collection of data (sometimes very large).  
- **Data point** — one specific piece of information in that collection.  
- **SQL** — create, query, and manage relational data (investigations, reporting, log correlation in some environments).

---

## Operating systems (OS)

**Definition:** Software layer between **hardware** and the **user**.

**Examples:** Linux®, macOS®, Windows® — different features and workflows.

### Linux (course emphasis)

- **Open source** — source code is public; community can contribute improvements.  
- **Not** a programming language.  
- Often administered via **commands** in a **command-line interface (CLI)**.  
- Deeper topics (kernel, GNU) appear in **later courses**.

---

## Web vulnerabilities

**Web vulnerability** — flaw in a web application a threat actor can abuse with malicious code or behavior to get **unauthorized access**, steal data, or deploy **malware**.

**Ongoing learning:** **OWASP Top 10** — industry reference for critical web application risks (review periodically).

---

## Antivirus software

- Prevents, detects, and removes **malware** and **viruses**.  
- Also called **anti-malware**.  
- May scan **memory** for **signatures** or suspicious patterns depending on product.

---

## Intrusion detection system (IDS)

- Monitors activity and **alerts** on possible intrusions.  
- Inspects **network packets** (small units of network data) to spot threats to sensitive data.  
- May also relate to **theft** or **unauthorized access** patterns.

---

## Encryption

**Goal:** Keep data **confidential** from anyone without authorization.

- Converts **plaintext** (readable) into **ciphertext** (scrambled/encoded form that should require proper keys/process to read).  
- **Cryptographic encoding** (in the encryption sense) protects against casual reading; strength depends on algorithm and key management.

**Course note:** **Encoding ≠ encryption.**

- **Encoding** — reversible transformation so **different systems** can exchange data (often **known/public** scheme).  
- **Encryption** — aims at **confidentiality** with cryptographic keys and proper design.

---

## Penetration testing (pen testing)

Simulated **authorized** attack to find **weaknesses** in systems, networks, sites, apps, and **processes**.

- Thorough **risk assessment** covering **external and internal** perspectives.  
- Identifies **threats** and **vulnerabilities** before real attackers do.
