# Communicate through a shell

The **shell** is the **command-line interpreter**. You can think of a shell as a translator between you and the computer system: shells let you give **commands** to the computer and receive **responses** from it.

When you enter a command into a shell, the shell runs internal processes to interpret your command, send it to the **kernel**, and return results.

---

## Types of shells

Linux includes many different shells, such as:

- **Bourne-Again Shell (bash)**
- **C Shell (csh)**
- **Korn Shell (ksh)**
- **Enhanced C shell (tcsh)**
- **Z Shell (zsh)**

All Linux shells use common Linux commands, but they can differ in features. For example, **ksh** and **bash** often use the dollar sign (`$`) to indicate where users type commands, while other shells—such as **zsh**—may use the percent sign (`%`).

---

## Bash

**Bash** is the **default** shell in most Linux distributions and is often considered **user-friendly**. You can use bash for basic Linux commands as well as larger projects.

Bash is also the most popular shell in the cybersecurity profession. You’ll use bash throughout this course as you learn and practice Linux commands.

---

## Key takeaways

- Shells are a fundamental part of the Linux operating system.  
- Shells let you give commands to the computer and receive responses.  
- Shells can be thought of as a translator between you and your computer system.  
- Many types of shells exist, but **bash** is the most commonly used shell in the cybersecurity profession.  
- You’ll learn how to enter Linux commands through the bash shell later in this course.
